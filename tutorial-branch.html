Tutorial Git - Trabalhando com Commits e Branches

Este guia prático ensina como trabalhar eficientemente com commits e branches no Git, duas funcionalidades essenciais para desenvolvimento colaborativo.

Trabalhando com Commits

Um commit representa um conjunto de alterações que você fez no código. É como um "ponto de salvamento" do seu trabalho.

Como fazer commits

  1. Verificar o status dos arquivos modificados:

    git status

  2. Adicionar arquivos para o próximo commit:

    # Adicionar arquivo específico
git add nome-do-arquivo.extensao

# Adicionar todos os arquivos modificados
git add .

# Adicionar múltiplos arquivos
git add arquivo1.txt arquivo2.js pasta/arquivo3.css

  3. Criar o commit com uma mensagem descritiva:

    git commit -m "Descrição clara das alterações feitas"

  4. Enviar os commits para o repositório remoto:

    git push origin nome-do-branch

Boas práticas para commits

  • Faça commits pequenos e frequentes, focados em uma única alteração lógica
  • Escreva mensagens de commit claras e descritivas
  • Use verbos no imperativo presente: "Adiciona", "Corrige", "Atualiza" (não "Adicionado", "Corrigido")
  • Estrutura recomendada para mensagens: 
    Resumo curto (até 50 caracteres)

Descrição mais detalhada explicando o "por quê" da mudança,
não o "como" (já que o código mostra isso).

Dica: Use git commit -am "mensagem" para adicionar e fazer commit de todos os arquivos modificados (não funciona para arquivos novos ainda não rastreados).

Visualizar histórico de commits

# Ver histórico simples
git log

# Ver histórico resumido e visualmente formatado
git log --oneline --graph --decorate

# Ver histórico de um arquivo específico
git log -- nome-do-arquivo.extensao

Trabalhando com Branches

Branches são "ramificações" independentes do código que permitem desenvolver funcionalidades sem afetar o código principal.

Gerenciando branches

  1. Listar branches existentes:

    # Listar branches locais
git branch

# Listar todos os branches (locais e remotos)
git branch -a

  2. Criar um novo branch:

    # Criar branch
git branch nome-do-novo-branch

# Criar e já mudar para o novo branch
git checkout -b nome-do-novo-branch

    No Git mais recente, você também pode usar:

    git switch -c nome-do-novo-branch

  3. Mudar para outro branch:

    # Método tradicional
git checkout nome-do-branch

# Método mais recente
git switch nome-do-branch

  4. Enviar branch para o repositório remoto:

    git push -u origin nome-do-branch

    O parâmetro -u (ou --set-upstream) configura o rastreamento entre o branch local e o remoto.

  5. Excluir branch:

    # Excluir branch local
git branch -d nome-do-branch

# Excluir branch local (forçado)
git branch -D nome-do-branch

# Excluir branch remoto
git push origin --delete nome-do-branch

Integrando branches

Método 1: Merge

O merge combina as alterações de um branch com outro:

# Mude para o branch que receberá as alterações (geralmente main/master)
git checkout main

# Faça o merge do outro branch para o atual
git merge nome-do-branch-a-integrar

Atenção: O merge pode gerar conflitos se o mesmo arquivo tiver sido alterado em ambos os branches. Nesse caso, você precisará resolver manualmente os conflitos.

Método 2: Rebase

O rebase reaplica seus commits sobre outra base, criando um histórico mais linear:

# Estando no seu branch de trabalho
git rebase main

# Depois, vá para o main e faça um merge (fast-forward)
git checkout main
git merge nome-do-branch

Cuidado: Nunca faça rebase de branches públicos/compartilhados, apenas de branches locais que ainda não foram enviados ao repositório remoto.

Fluxo de trabalho com branches

Um fluxo de trabalho comum com Git:

  1. Atualize seu branch principal: git checkout main && git pull
  2. Crie um branch para sua tarefa: git checkout -b feature/nova-funcionalidade
  3. Trabalhe, faça commits: git add . && git commit -m "Implementa funcionalidade X"
  4. Mantenha seu branch atualizado: git fetch && git rebase origin/main
  5. Envie seu branch para o remoto: git push -u origin feature/nova-funcionalidade
  6. Crie um Pull Request no GitHub ou plataforma similar
  7. Após aprovação, faça o merge (na plataforma ou localmente)

Resolvendo Conflitos

Conflitos ocorrem quando o Git não consegue integrar automaticamente alterações de diferentes branches.

Passos para resolver conflitos:

  1. O Git marcará os arquivos com conflitos. Abra-os e você verá algo como:

    <<<<<<< HEAD
código da versão atual
=======
código da outra versão
>>>>>>> nome-do-branch
  2. Edite o arquivo para resolver o conflito (escolha uma versão ou combine ambas)
  3. Remova os marcadores (<<<<<<<, =======, >>>>>>>)
  4. Adicione o arquivo resolvido: git add nome-do-arquivo
  5. Continue o merge ou rebase: 
    # Se estiver em um merge
git commit

# Se estiver em um rebase
git rebase --continue

Dica: Use ferramentas visuais para resolver conflitos mais facilmente: git mergetool ou IDEs como VSCode, IntelliJ, etc.

Comandos Git adicionais úteis

  • Ver diferenças: git diff
  • Desfazer modificações: git checkout -- arquivo ou git restore arquivo
  • Desfazer commits: git reset HEAD~1 (mantém alterações) ou git reset --hard HEAD~1 (descarta alterações)
  • Salvar trabalho temporariamente: git stash e git stash pop
  • Ver configurações: git config --list

Conclusão

Dominar commits e branches é essencial para trabalhar eficientemente com Git. Pratique regularmente esses comandos e você se tornará mais confiante no controle de versão de seus projetos. Lembre-se que o Git foi projetado para facilitar o trabalho colaborativo e proteger seu código, então aproveite todos os seus recursos!
